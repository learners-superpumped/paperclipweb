## 발견된 버그

### BUG-1 [CRITICAL] 회원가입/로그인 서버 에러
- **플로우-단계:** B-4 (회원가입 제출)
- **재현:** /signup 에서 이메일 입력 후 "Create Account" 클릭
- **예상:** 매직 링크 이메일 발송 후 "이메일을 확인하세요" 메시지 표시
- **실제:** "Server error - There is a problem with the server configuration" 전체 화면 에러
- **영향:** 신규 유저 온보딩 완전 차단. 서비스 이용 불가.
- **추정 원인:** Vercel 배포 환경에 NextAuth 필수 환경변수 미설정 (`AUTH_SECRET`, `NEXTAUTH_URL`, `AUTH_RESEND_KEY` 또는 이메일 제공자 설정 등)

### BUG-2 [HIGH] /api/webhooks/stripe 경로 404
- **플로우-단계:** D (API 헬스 체크)
- **재현:** POST https://paperclipweb-learners.vercel.app/api/webhooks/stripe
- **예상:** 400 (invalid signature)
- **실제:** 404 Not Found
- **영향:** Stripe Webhook이 spec 경로(/api/webhooks/stripe)로 설정된 경우 결제 이벤트 미수신. 실제 구현은 /api/stripe/webhook 경로 사용.
- **권고:** Stripe 대시보드 webhook URL을 /api/stripe/webhook 으로 설정하거나, 코드에서 경로 통일 필요.

### BUG-3 [MEDIUM] 스펙 대비 플랜명 불일치
- **설명:** 스펙에는 "Free / Pro $19/mo / Business $49/mo"로 정의되어 있으나 실제 랜딩 페이지는 "Free / Starter $19 / Pro $49" 표시
- **영향:** 마케팅 문서/스펙과 실제 서비스 간 불일치. PMF 검증 시 혼란 가능.

### BUG-4 [LOW] 모바일 서브헤드라인 텍스트 공백 누락
- **플로우-단계:** E (반응형 - 375px)
- **재현:** 모바일(375px)에서 랜딩 히어로 섹션 서브헤드라인 확인
- **예상:** "No API keys needed. Start running agents instantly with bundled credits."
- **실제:** "No API keys needed.Start running agents instantly with bundled credits." (공백 없음)

### BUG-5 [MEDIUM] 스펙 대비 인증 방식 불일치
- **설명:** 스펙(Part 3 기술 스택)에는 "GitHub, Google 소셜 로그인"으로 명시되어 있으나 실제 구현은 이메일 매직 링크만 제공
- **영향:** 스펙과 구현 간 의도적 변경으로 보이나, 공식 스펙 업데이트 필요
